# 03 - Type Names and Meme System

## Goal

Make type outputs memorable and shareable while preserving a serious secondary interpretation.

## Naming Structure

For each type:

- `Primary name` (memeable)
- `Secondary label` (more reflective/neutral)
- `1-line descriptor`

Example format:

- `CKVD` → "Dream Client" / "Collaborative Vision Builder"
- Descriptor: "Clear asks, kind tone, big ideas, and trusts the agent to explore."

## Naming Principles

- 2-4 words max
- Easy to screenshot and repost
- No cruel labels
- Keep roast level playful

## Output Template

- "Your type: `CKVD`"
- "Nickname: Dream Client"
- "Serious read: Collaborative Vision Builder"

## Why This Matters

Users share identity labels, not scoring formulas. The nickname is the viral payload.
